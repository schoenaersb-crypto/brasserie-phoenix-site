# Agent 03 — qa

## Rôle
Contrôle qualité automatisé du site personnalisé **avant** tout déploiement.
C'est le garde-fou du pipeline : il refuse de laisser passer un site incomplet.

## Entrée
- `clients/<slug>/site/` (produit par 02-personalizer).

## Sortie
- `clients/<slug>/rapports/qa.md` : rapport détaillé PASS / FAIL.
- Verdict : **PASS** (feu vert déploiement) ou **FAIL** (retour à 02-personalizer).

## Contrôles

### 1. Anti-crochets (critère bloquant)
Rechercher toute occurrence de `[` restante dans `site/index.html` :
```bash
grep -n '\[' clients/<slug>/site/index.html
```
- **Critère de succès : 0 crochet restant.** Toute occurrence = FAIL.

### 2. Vérification des chemins
- Aucun `../../../` ne doit subsister.
- `_base/assets/css/base.css` et `_base/assets/js/main.js` existent bien sous `site/_base/`.
- Les `href`/`src` internes pointent vers des fichiers réellement présents.
```bash
grep -n '\.\./\.\./\.\.' clients/<slug>/site/index.html   # doit être vide
test -f clients/<slug>/site/_base/assets/css/base.css
test -f clients/<slug>/site/_base/assets/js/main.js
```

### 3. Validation HTML
- Structure : `<!doctype html>`, `<html lang="fr">`, `<head>`, `<body>` présents et fermés.
- Balises correctement fermées, pas de balise orpheline.
- `<title>` et `<meta name="description">` renseignés (non vides, sans crochet).
- Attributs `alt` présents sur les images.

### 4. Liens tel: / mailto:
- Chaque `tel:` est **sans espaces** et au format international (`tel:+3221112233`).
- Chaque `mailto:` contient une adresse email valide (`user@domaine.tld`).
- L'URL Instagram (`href`) est une URL absolue `https://`.
```bash
grep -oE 'tel:[^"]+' clients/<slug>/site/index.html
grep -oE 'mailto:[^"]+' clients/<slug>/site/index.html
```

### 5. Cohérence identité visuelle
- `:root {}` contient des couleurs hexadécimales valides (pas de placeholder).
- L'URL Google Fonts référence bien les deux polices du brief.

## Format du rapport `qa.md`

```markdown
# Rapport QA — test-bar-la-playa
Date : 2026-07-04

| Contrôle              | Résultat | Détail                     |
|-----------------------|----------|----------------------------|
| Crochets restants     | ✅ 0     | —                          |
| Chemins _base/        | ✅ PASS  | ../../../ absent           |
| Validation HTML       | ✅ PASS  | structure conforme         |
| Liens tel:/mailto:    | ✅ PASS  | 3 tel, 2 mailto valides    |
| Identité visuelle     | ✅ PASS  | #C0392B / #F1C40F          |

VERDICT : ✅ PASS — déploiement autorisé.
```

## Critère de succès global
**PASS uniquement si les 5 contrôles passent ET qu'il reste 0 crochet.**
Sinon → **FAIL**, le rapport liste les défauts et le pipeline repasse par 02-personalizer.

## Passage de relais
- **PASS** → transmet le feu vert à l'agent **04-deployer**.
- **FAIL** → renvoie la liste des défauts à l'agent **02-personalizer** ; aucun déploiement.
